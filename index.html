The system relies on a base-3 (trinary) foundation, augmented by a transposition layer and a recursive compression matrix.

2. Decryption Pipeline (The Four Layers)

To successfully extract plaintext from a numeric payload, the decoder must execute four distinct computational phases in strict sequence.

Phase I: The Scramble Phase (Transposition)

Identified through Java-based source code (ITZLOGHOTXD/Cipher.java), this layer introduces entropy by swapping positional integers before expansion. This prevents basic frequency analysis from identifying the underlying trinary tokens.

  • 14 ↔ 41
  • 25 ↔ 52
  • 36 ↔ 63

Phase II: The Unsquish Protocol (Decompression)

Redundant trinary pairs are collapsed into single placeholder integers (4, 5, and 6). Decryption requires a recursive expansion loop that continues until no compressed digits remain.

  • 4 → 11
  • 5 → 22

Phase III: The Trinary Matrix (Base-3 Substitution)

Tokens are parsed in discrete 3-digit segments. Standard English characters (A-Z) map sequentially from 111 to 332.

  • Functional Space: The digit 0 acts as a direct substitute for a whitespace null.
